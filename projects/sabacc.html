Overview

This is a fully networked, turn-based card game running inside a live 40-70 player Roblox server. Multiple tables run at once, players sit down at them together, there's a buy-in economy, and a shift-token ability layer that lets players bend the rules of a hand mid-round.

Problem

Card games need real server authority. You can't trust the client with hand state, full stop, but the game still has to feel instant to the players sitting at the table. So turn structure, draw resolution, and scoring all live server-side, and get replicated down through a state store the UI reacts to.

Architecture

Game state machine flow:

WAITING → VOTING → DEALING → PLAYING → REVEAL → IMPOSTOR_CHOICE? → SCORING → loop / GAME_OVER

A pure state module (GameStateModule) owns turn advancement, cycle resets, and active-player tracking, and it's completely decoupled from networking, so I can unit test game logic without a live server. The service layer wraps it, validates all client input at the boundary, and dispatches Rodux actions that replicate only to the players who actually need them.

Shift Tokens

This is a pluggable ability system: free draw, chip refunds, embezzlement, and a "major fraud" token that forces every Impostor card to a fixed value for the round. New tokens just register in one data table and automatically pick up backpack tools, hotbar slots, and Cmdr admin support.

Result

Stable multi-table concurrent play, with anonymous-identity masking for certain factions, a buy-in economy tied into the wider server economy, and zero desync issues across a full production season.
