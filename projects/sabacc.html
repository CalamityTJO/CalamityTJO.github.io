Overview

A fully networked, turn-based card game running inside a live 40–70 player Roblox server. Multiple simultaneous tables, seated multiplayer, buy-in economy, and a shift-token ability layer that lets players bend the rules of a hand mid-round.

Problem

Card games need strict, provably-fair server authority — the client can never be trusted with hand state — while still feeling instant to the players at the table. That means turn structure, draw resolution, and scoring all live server-side, replicated down through a state store the UI reacts to.

Architecture

Game state machine flow:

WAITING → VOTING → DEALING → PLAYING → REVEAL → IMPOSTOR_CHOICE? → SCORING → loop / GAME_OVER

A pure state module (GameStateModule) owns turn advancement, cycle resets, and active-player tracking, completely decoupled from networking — so game logic can be unit tested without a live server. The service layer wraps it, validates all client input at the boundary, and dispatches Rodux actions that replicate only to the players who need them.

Shift Tokens

A pluggable ability system — free draw, chip refunds, embezzlement, and a "major fraud" token that forces every Impostor card to a fixed value for the round. New tokens register in one data table and automatically pick up backpack tools, hotbar slots, and Cmdr admin support.

Result

Stable multi-table concurrent play with anonymous-identity masking for certain factions, buy-in economy tied into the wider server economy, and zero desync issues across a full production season.
