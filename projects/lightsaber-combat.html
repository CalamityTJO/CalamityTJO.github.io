Lightsaber Combat System

LuauCombatServer-Authoritative

Overview

I didn't build this framework from scratch, it was already in place when I joined the project, but I work in it regularly and I've made real changes to it. It's a request/response combat system driving lightsaber interactions: attacks, blocks, parries, and kill confirmation, sitting between client input and server-validated outcomes.

What I've worked on

Every combat action is a numbered request in CombatConstants.ENUM.REQUEST, and the server validates state (stamina, cooldowns, distance, target legitimacy) before ever trusting a client's claim that a hit landed. One thing I added recently: per-saber-type and per-alignment audio. Different saber types (single, dual, double, and so on) now carry their own ignite, swing, and clash sound sets instead of sharing one generic bucket, and specific crystal alignments can override that further, so a Sith single-blade saber sounds distinct from a Jedi one using the same base type. 

-- swing sound now resolves per saber type, with an alignment override on top
local swingSounds = if self.SithSingleBlade then Sounds.Single.Sith.Swing else Sounds[self.SoundIndex].Swing
local selectedSound = swingSounds[math.random(1, #swingSounds)]
