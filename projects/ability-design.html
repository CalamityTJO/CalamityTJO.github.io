Gameplay Ability Design

LuauGame DesignCombat

Overview

Beyond working in an existing combat framework, I've designed and built abilities inside it. Each one is a self-contained module with its own cooldown, targeting, validation, and VFX, plugged into a shared ability pipeline so the rest of the game doesn't need to know the specifics.

Abilities I've built

  • Force Pull: target acquisition, cooldown gating, and a capped number of targets yanked per cast so it can't be chained into a stun-lock.
  • Affliction: a debuff that slows a target and marks them, with the slow amount and regen penalty scaled by the caster's rank.
  • Force Bubble: a protective barricade with rank-scaled buffs, its own ignite/active animations, VFX, and caster-lock rules (you can't pop it while choked, stasised, or mind-tricked).

Why it matters

None of that skill is tied to Star Wars specifically. Cooldowns, targeting, rank/level scaling, VFX timing, and making sure an ability can't be spammed or abused are the same problems in any game with player abilities. I can take that same approach into a different theme without missing a step.

Result

These abilities are live and balanced in a production game with real player feedback shaping them over time, not just prototypes.
