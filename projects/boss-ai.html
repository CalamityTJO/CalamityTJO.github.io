AI Boss/Minion Combat

LuauAICombatComponent-Based

Overview

A phase-driven NPC AI system for raid-style boss encounters — bosses cycle through skill phases, spawn and coordinate minions, and equip dynamically-built lightsaber loadouts, all running server-side against the same combat framework real players use.

Problem

Boss fights need to feel scripted and readable to players (clear phase telegraphs, fair timing windows) while staying entirely server-driven — no client-side animation can be trusted to gate damage. The AI also has to reuse the existing player combat pipeline rather than fork a separate one, so any lightsaber or Force-power balance change applies to bosses for free.

Architecture

Tag-based server components drive each boss instance. A phase controller advances through a skill list on timers and health thresholds, calling into the same Lightsaber signal/combat request handlers a real player's client would trigger — the boss is, from the combat system's perspective, just another combatant with a scripted "player". Saber builds come from the same CustomisationUtil pipeline used for player character customization, so new boss loadouts require no new code, just data.

Minion Coordination

Minions spawn on boss-phase triggers and register with the boss component via Janitor-managed connections, so a boss death or phase transition cleanly tears down every minion it owns without manual bookkeeping per spawn.

Result

Reused 100% of the existing combat/damage pipeline for AI encounters — no parallel "NPC combat" system to maintain — while keeping per-encounter authoring down to a data file plus a phase list.
