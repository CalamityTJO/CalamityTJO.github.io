AI Boss/Minion Combat

LuauAICombatComponent-Based

Overview

A phase-driven NPC AI system for raid-style boss encounters. Bosses cycle through skill phases, spawn and coordinate minions, and equip lightsaber loadouts built dynamically, all running server-side against the exact same combat framework real players use.

Problem

Boss fights need to feel scripted and readable to players, with clear phase telegraphs and fair timing windows, while staying entirely server-driven since no client-side animation can be trusted to gate damage. The AI also had to reuse the existing player combat pipeline instead of forking a separate one, so any lightsaber or Force-power balance change applies to bosses for free.

Architecture

Tag-based server components drive each boss instance. A phase controller advances through a skill list on timers and health thresholds, calling into the same Lightsaber signal/combat request handlers a real player's client would trigger. From the combat system's point of view, the boss is just another combatant with a scripted "player" behind it. Saber builds come from the same CustomisationUtil pipeline used for player character customization, so new boss loadouts need no new code, just data.

Minion Coordination

Minions spawn on boss-phase triggers and register with the boss component through Janitor-managed connections, so a boss death or phase transition cleanly tears down every minion it owns without any manual bookkeeping per spawn.

Result

Reused the entire existing combat/damage pipeline for AI encounters instead of maintaining a parallel "NPC combat" system, and kept per-encounter authoring down to a data file plus a phase list.
